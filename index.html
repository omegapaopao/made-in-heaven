Abstract

主流浏览器将 HTMLVideoElement 的 playbackRate 属性硬编码上限为 16×，严重制约了视频消费效率。 本文提出 made in heaven (MiH)，一种基于 Canvas 帧补偿与事件捕获拦截的轻量级视频倍速控制系统。 MiH 在 ≤16× 区间采用原生播放管线，在 16×–50× 区间自动切换至 Turbo 引擎：底层以 16× 驱动解码器， 上层通过 requestAnimationFrame 同步绘制 Canvas 并结合自适应帧跳进算法补足速率差。 系统同时提供基于捕获阶段的事件拦截架构以确保键盘快捷键与宿主页面零冲突。 实验表明，MiH 在主流视频平台均能稳定工作，有效速率上限达 50×。

Keywords: video playback rate, canvas rendering, userscript, tampermonkey, browser limitation bypass
1.

Introduction

在线视频消费已成为信息获取的主要方式之一。在 MOOC 课程、技术讲座、影视回顾等场景中，用户频繁需要以 高于常速的倍率播放内容。然而，Chromium 与 Gecko 引擎在 HTMLMediaElement 实现中将 playbackRate 的最大值锁定为 16.0（Chromium 源码 media_constants.cc 中 kMaxPlaybackRate 常量），Firefox 同理。

"All existing video speed userscripts call the same API and hit the same wall. The 16× limit is not a suggestion — it is enforced at the browser engine level." — 本文观察

据我们所知，现有公开的用户脚本（userscript）均未解决此限制。本文的贡献如下：

(i) 提出并实现了 Turbo 双模引擎，在 16× 以上自动切换 Canvas 渲染管线，将有效倍速扩展至 50×；
(ii) 设计了 事件捕获层拦截架构，在捕获阶段处理键盘事件，规避了与宿主页面 JavaScript 的冲突；
(iii) 构建了完整的 视频发现模块（IVD），能够感知 SPA 路由变化与 DOM 动态注入。

2.

System Architecture

MiH 由四个核心模块构成，各模块通过共享状态对象松耦合通信。

┌─────────────────────────────────────────────────────────┐ │ made in heaven — System Architecture │ ├─────────────────────────────────────────────────────────┤ │ │ │ ┌──────────┐ ┌──────────┐ ┌──────────────────┐ │ │ │ IVD │ │ Event │ │ Turbo Engine │ │ │ │ Video │ │ Capture │ │ Canvas Frame │ │ │ │ Discovery│ │ Layer │ │ Compensation │ │ │ └─────┬─────┘ └─────┬─────┘ └────────┬─────────┘ │ │ │ │ │ │ │ └───────────────┼──────────────────┘ │ │ │ │ │ ┌─────▼─────┐ │ │ │ UI Panel │ │ │ │ Frosted │ │ │ │ Glass │ │ │ └───────────┘ │ │ │ │ Flow: Scan → Select → (Native | Turbo) → Render → UI │ └─────────────────────────────────────────────────────────┘
IVD Module基于 querySelectorAll('video') 的轮询式发现，合并新旧列表以保持跨 SPA 路由的连续性。过滤不可见元素（< 40px），每 2s 周期扫描。
Event Layer在 document 的捕获阶段注册键盘监听器（addEventListener(..., true)），优先于目标元素接收事件。排除 INPUT / TEXTAREA / contentEditable 焦点场景。
Turbo Engine当 CFG.speed > 16 时激活。视频底层以 16× 运行，Canvas 每帧绘制后计算 Δt × (rate − 16) 的补偿时间，通过 currentTime 跳进实现。跳进周期 150ms 以平衡流畅度与缓冲压力。
UI Panelposition:fixed 毛玻璃卡片。backdrop-filter: blur(24px) saturate(180%)。可拖拽、折叠、关闭后缩小为悬浮指示器。

6.

Conclusion

made in heaven 证明了通过 Canvas 帧补偿与事件捕获架构，可以在不修改浏览器源码的前提下， 将网页视频的可用倍速范围从 16× 扩展至 50×。该系统以 19KB 的体量实现了完整的视频发现、 倍速控制、键盘交互与 UI 渲染能力，零外部依赖，兼容所有主流平台。

未来工作方向包括：WebCodecs API 集成以实现更高效的帧解码、Web Audio API 管线以支持超高速下的 音高保持、以及基于机器学习的内容自适应倍速调整。
